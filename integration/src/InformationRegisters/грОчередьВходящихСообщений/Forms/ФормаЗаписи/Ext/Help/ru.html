Запись регистра сведений "Очередь входящих сообщений" предназначена для хранения данных входящего сообщения, поступившего в систему от адаптера ESB, и сведений об этом входящем сообщении.

В форме записи отображается:

  • Идентификатор сообщения - уникальный идентификатор сообщения, поступившего в систему из ESB (тип - Уникальный идентификатор);
  • Хранилище - сами данные, передаваемые в сообщении (тип - Хранилище значения);
  • Дата регистрации - дата, когда входящее сообщение было зарегистрировано в очереди (тип - Дата);
  • Формат сообщения - формат, в котором записано входящее сообщение (тип - перечисление "Форматы сообщений": XML, JSON, FastInfoset);
  • Класс сообщения - класс, присвоенный сообщению при его поступлении в ESB для передачи (тип - Строка);
  • Дата создания - дата, когда входящее сообщение было создано (тип - Дата).
  • Статус - текущий статус входящего сообщения (тип - перечисление "Статусы соощений"). Может принимать значения:

Новое - сообщение только что поступило в систему;

Ожидание обработки - сообщение ожидает обработки;

В обработке - сообщение находится в обработке;

Отсутствует обработчик - для данного класса сообщения не найден обработчик;

Обработано - сообщение обработано;

Ошибка обработки - при выполнении обработчика возникла ошибка;

Обработка отменена - обработка сообщения отменена;

Ожидание отправки - сообщение ожидает отправки (для исходящих сообщений);

Отправлено - сообщение отправлено (для исходящих сообщений);

Отправка отменена - отправка сообщения отменена (для исходящих сообщений).
